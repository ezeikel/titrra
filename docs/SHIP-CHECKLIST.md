## 🔌 On-device LOCAL API testing (cleartext) — 2026-06-22

To test the mobile app against a LOCAL web API on the emulator: (1) start web (`pnpm dev:web` → localhost:3000), (2) `adb -s emulator-5554 reverse tcp:3000 tcp:3000` (maps emulator localhost:3000 → host), (3) build with `EXPO_PUBLIC_API_URL=http://localhost:3000`. CAVEAT: Android release builds BLOCK cleartext http by default → the request silently fails. FIX (added to app.config.ts): `expo-build-properties.android.usesCleartextTraffic` = `env !== 'production'` (a derived `allowCleartext`), so dev + preview builds permit http (localhost/LAN/staging) but PRODUCTION stays https-only. Needs a `prebuild --clean` to regenerate the manifest. Build the **preview** variant (cleartext on, bundle `...app.internal`) for this — launch `com.chewybytes.titrra.app.internal`, and re-assert `adb reverse` after each install. (Reusable for every app — add to the playbook.)
